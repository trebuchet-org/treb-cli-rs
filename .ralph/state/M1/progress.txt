## Codebase Patterns
- Fork command module: `crates/treb-cli/src/commands/fork.rs` contains all fork subcommands
- Per-module `styled()` helper: each command module defines its own `fn styled(text: &str, style: Style) -> String` that checks `color::is_color_enabled()`
- Table building pattern: `output::build_table(&headers)` → `table.add_row(vec![...])` → `output::print_table(&table)`
- Color constants: `color::NAMESPACE` (cyan bold), `color::SUCCESS` (green bold), `color::ERROR` (red bold), `color::CHAIN` (magenta bold), `color::LABEL` (bold), `color::STAGE` (cyan bold), `color::WARNING` (yellow), `color::MUTED` (dimmed) in `crates/treb-cli/src/ui/color.rs`
- History action→style mapping: enter=SUCCESS, exit=WARNING, revert=MUTED, restart=STAGE
- Diff change→style mapping: added=green, removed=red, modified=yellow (inline `Style::new()` not color constants)
- Golden file tests: use `IntegrationTest` builder with `.extra_normalizer()` for dynamic content; `UPDATE_GOLDEN=1` to regenerate
- Uptime normalization: use `UptimeNormalizer` for tests with `format_uptime()` output in golden files
- Fork deployments: count keys starting with `fork/` in `deployments.json` via `load_json_map()`
