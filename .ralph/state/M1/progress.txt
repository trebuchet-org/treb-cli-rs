## Codebase Patterns
- Shorthand verifier flags (--etherscan, --blockscout, --sourcify) are resolved in main.rs match arm, not in verify.rs
- verify.rs run() and run_batch() accept &[String] for verifiers; the caller in main.rs builds the Vec
- CLI integration tests use `cargo_bin_cmd!("treb-cli")` macro via `assert_cmd` crate
- Test helpers `init_project_with_deployments` and `init_project_with_verified_deployments` in cli_verify.rs set up fixture data from treb-core fixtures
- All verify tests pass with `cargo test -p treb-cli -- verify` (covers both cli_verify.rs and integration_verify.rs)
- `std::env::set_var` and `std::env::remove_var` require `unsafe {}` blocks in Rust 2024 edition
- `ui::badge::verification_badge()` / `verification_badge_styled()` for compact per-verifier status display (e.g. `e[V] s[X] b[-]`)
- Use `styled()` helper in verify.rs for all color-gated text; progress to stderr, results to stdout
