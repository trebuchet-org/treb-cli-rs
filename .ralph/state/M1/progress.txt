## Codebase Patterns
- Compose result structs (`ComponentResultEntry`, `ComposeTotals`) now use `#[serde(rename_all = "camelCase")]` — JSON keys are camelCase (e.g. `gasUsed`, `notExecuted`)
- `PipelineResult.gas_used` already exists in `treb-forge/src/pipeline/types.rs` — no need to add it
- Compose unit tests live in `compose.rs` itself under `#[cfg(test)] mod tests`; run with `cargo test -p treb-cli --bin treb-cli -- compose`
- 6 pre-existing golden file failures (`compose_dry_run_*`) are from US-002 warning banner + US-001 camelCase changes — deferred to P7-US-007
- `ComponentResultEntry` is constructed in 4 places in the execution loop: skipped, not-executed, success (from pipeline result), and failed
- Golden file integration tests capture stdout+stderr combined — any new stderr output (e.g. stage indicators, banners) will break golden files
- `output::print_warning_banner()` goes to stdout; for stderr use `eprintln!("{}", output::format_warning_banner(...))`
- Stage indicator emojis: rocket(\u{1f680})=start, hammer(\u{1f528})=executing, checkmark(\u{2705})=complete, skip(\u{23ed}\u{fe0f})=skipped, cross(\u{274c})=failed
- `ScriptConfig::to_forge_command()` returns `Vec<String>` of forge CLI args — use for `--dump-command` per-component output
- When adding new CLI flags to compose: update Compose enum variant in main.rs, the match arm destructuring, AND compose::run() signature
- `styled()` helper pattern: `fn styled(text: &str, style: Style) -> String` gated on `color::is_color_enabled()` — used in show.rs, verify.rs, and now compose.rs
- Color styles: SUCCESS=green+bold, ERROR=red+bold, WARNING=yellow, MUTED=dimmed, LABEL=bold
- `run::resolve_rpc_url_for_chain_id()` and `run::fetch_chain_id()` are `pub(crate)` — reusable for chain ID resolution from any command
