# Ralph Progress Log
## Codebase Patterns
- When a clap flag becomes global in `crates/treb-cli/src/main.rs`, mirror it in `crates/treb-cli/build.rs` and refresh the affected `tests/golden/help_*` snapshots because the option propagates into root and subcommand help.
- When a command relies on `ui::selector` or destructive prompt-skipping logic, pass `Cli.non_interactive` all the way down; `is_non_interactive(false)` only honors env/TTY fallbacks and silently ignores the parsed global flag.
- `crates/treb-cli/src/ui/interactive.rs` treats `TREB_NON_INTERACTIVE=1|true` as opt-in, while `CI` remains a separate `true`-only switch; keep those env semantics distinct when refactoring prompt detection.
- When adding subcommand short flags in `crates/treb-cli/src/main.rs`, mirror them in `crates/treb-cli/build.rs`; fast regression coverage can live in `src/main.rs` via `parse_cli_from(...)` and targeted subcommand help assertions without introducing new golden snapshots.
