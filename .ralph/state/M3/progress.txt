# Ralph Progress Log
## Codebase Patterns
- Registry store files should use `read_versioned_file()` for backward-compatible reads and `write_versioned_file()` for locked atomic writes to the `{"_format":"treb-v1","entries":...}` wrapper format.
- Map-backed registry stores should sort entries into a `BTreeMap` before `write_versioned_file()` so wrapped JSON stays deterministic for tests and diffs.
- `ForkStateStore` should persist through a sorted wrapper view of `forks` while keeping the `history` vector order intact, so `fork.json` remains deterministic without changing recency semantics.
- Secondary-index stores that persist `Vec<String>` ID lists should sort those vectors before saving or returning rebuilt data, otherwise hash-map iteration can make the JSON output nondeterministic.
- Registry-only changes should be validated with `cargo check -p treb-registry` and `cargo test -p treb-registry` before running broader workspace checks.
