## Codebase Patterns
- For CLI flag changes in `crates/treb-cli`, keep the derive parser in `src/main.rs`, the shell completion builder in `build.rs`, and a focused `parse_cli_from(...)` regression test in sync.
- For scoped deployment queries in CLI commands, filter `Registry::list_deployments()` with `commands::list::filter_deployments()` and resolve the user query with `commands::resolve::resolve_deployment_in_scope()` so namespace/network/fork filters affect ambiguity resolution consistently.
- For scoped commands that support interactive deployment selection, pass the filtered deployment slice into `fuzzy_select_deployment_id()` too; otherwise the prompt can offer out-of-scope deployments even when typed queries are scoped correctly.
- For clap help changes that are specific to one command, add or refresh a dedicated `help_<command>` golden in `crates/treb-cli/tests/integration_help.rs`; root `treb --help` snapshots do not catch all subcommand help drift.
- For scoped CLI behavior in `crates/treb-cli/tests/integration_*.rs`, prefer direct `TestContext` subprocess assertions plus ad hoc `Registry::insert_deployment()` setup when the shared fixtures do not include the ambiguous namespace/network combinations you need; keep goldens for formatting-heavy coverage.
