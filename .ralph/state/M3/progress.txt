- `write_versioned_file()` now writes bare JSON for Go compatibility, while `read_versioned_file()` and `read_versioned_file_compat()` must keep accepting legacy `{"_format":"treb-v1","entries":...}` payloads; persistence tests should assert the top-level bare object directly.

- CLI golden snapshots of persisted `.treb/*.json` artifacts must also use the bare-map registry format; if an older golden still contains `_format`/`entries`, normalize it to `.entries // .` before relying on the failure diff.
