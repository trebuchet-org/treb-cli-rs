## Codebase Patterns
- Nested map registry stores should sort both outer and inner maps into `BTreeMap`s before `write_versioned_file()` so bare JSON output stays deterministic.
- Top-level CLI commands must stay in sync across `crates/treb-cli/src/main.rs`, `crates/treb-cli/build.rs`, and any manually maintained root-help goldens because clap parsing, shell completions, and grouped help are wired separately.
- Addressbook subcommands should reuse `resolve_effective_chain_id()` in `crates/treb-cli/src/commands/addressbook.rs` so numeric chain IDs, Alloy named chains, and config defaults resolve consistently across `set`, `remove`, and `list`.
- Default CLI subcommands implemented through `normalize_cli_args()` should insert after parent-scoped flags like `-n/-s` and before subcommand-only flags like `--json`, while still leaving `--help` unnormalized.
- Addressbook CLI integration coverage is stable when it uses `TestContext::new("project")`, runs `treb init`, passes a numeric `--network`, and sets `NO_COLOR=1` for human-output assertions.
- Stateful alias-compatibility tests should compare each command in its own identically seeded temp project; seed registry-backed state up front so mutations like `set` or `remove` do not leak across invocations.
