## Codebase Patterns
- When changing CLI command names or nesting in `crates/treb-cli/src/main.rs`, also update `build_grouped_help()` there and `crates/treb-cli/build.rs`; root help and generated shell completions are maintained separately from the derive parser.
- When renaming a CLI command that has golden coverage, update both the `crates/treb-cli/tests/integration_*.rs` command argv and the matching `crates/treb-cli/tests/golden/*/commands.golden` header, because the harness snapshots the invoked argv verbatim.
- For spelling-only CLI renames that need backward compatibility, keep the canonical subcommand name in `src/main.rs` and add the old spelling as a hidden clap alias there and in `build.rs`; that preserves parsing, root help, and generated shell completions without a second runtime handler.
- For a command that should default to an existing subcommand without changing clap help/completion structure, normalize argv in `crates/treb-cli/src/main.rs` before clap parsing and skip that rewrite when `--help` is present.
- Help regressions need coverage in both `treb --help` and subcommand `--help` snapshots: root help is custom-built in `build_grouped_help()`, while subcommand help still comes from clap and can drift independently.
- For CLI renames that must preserve legacy entrypoints, keep one focused binary suite in `crates/treb-cli/tests/cli_compatibility_aliases.rs` that compares canonical and compatibility stdout directly; use feature-specific suites for deeper behavior, not for alias parity alone.
