## Codebase Patterns
- Use `serde(rename = "...")` on enum variants for exact Go-matching JSON strings
- ProposalStatus uses lowercase Go constants ("pending", "active"), all other enums use UPPERCASE
- Newtypes for IDs use `#[serde(transparent)]` for seamless JSON serialization
- Workspace dependencies are centralized in root Cargo.toml; crates use `{ workspace = true }`
- The primitives module in lib.rs re-exports alloy types and defines type aliases (ChainId, TxHash)
- Always run `cargo fmt` after writing code — rustfmt.toml merges imports into crate-level groups
- Use `#[serde(rename_all = "camelCase")]` on structs for Go-compatible JSON; override with `#[serde(rename = "...")]` for fields like `type` and `chainId` that conflict with Rust keywords or camelCase rules
- Go `omitempty` on strings → `#[serde(default, skip_serializing_if = "String::is_empty")]`; on `*T` → `Option<T>` with `skip_serializing_if = "Option::is_none"`; on maps → `HashMap` with `skip_serializing_if = "HashMap::is_empty"`
- Go nil slices serialize as `null`, not `[]` — use `Option<Vec<T>>` in Rust for nullable slices; use bare `Vec<T>` for slices that should always be `[]` when empty
- Go `omitempty` on `uint64` skips zero — use `#[serde(default, skip_serializing_if = "is_zero_u64")]` with a helper fn
- Go `map[string]any` → `HashMap<String, serde_json::Value>` in Rust
- Golden file fixtures live at `crates/treb-core/tests/fixtures/`; timestamps are normalized to UTC (Z suffix) for round-trip compatibility with chrono's DateTime<Utc>
- Go-generated registry data lives at `/home/sol/projects/treb-state/.treb/` (deployments.json, transactions.json, safe-txs.json)
