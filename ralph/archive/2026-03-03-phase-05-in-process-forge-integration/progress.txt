## Codebase Patterns
- All crate Cargo.toml files use `workspace = true` for package metadata (version, edition, rust-version, authors, license)
- All external dependencies are declared in root `[workspace.dependencies]` and referenced with `{ workspace = true }`
- lib.rs uses `//!` doc comments, `pub mod` declarations, and `pub use` re-exports of key types
- Errors use `TrebError` enum from `treb-core::error` with `thiserror` derive â€” variants: Config, Registry, Forge, Io
- `treb-core::Result<T>` is the standard result alias
- Tests live in `#[cfg(test)] mod tests` blocks within each module
- Foundry crates (v1.5.1) require solar crate patches â€” see `[patch.crates-io]` in root Cargo.toml
- Foundry uses `eyre::Result` internally; convert to `TrebError::Forge` at API boundaries
- `foundry_compilers::ProjectCompileOutput` does NOT implement `Debug` â€” cannot use `unwrap_err()` on `Result<CompilationOutput, _>` in tests; use `match` instead
- `foundry_config::Config::load_with_root(path)` loads config from a directory; `config.project()` returns `Project<MultiCompiler>`
- `project.paths.has_input_files()` checks for source files; `project.compile()` returns `ProjectCompileOutput`
- `foundry-compilers` (v0.19.14) is from crates.io, not from the foundry git repo
- `ScriptArgs::default()` gives Rust defaults (empty sig, 0 gas multiplier, 0 batch_size) â€” clap `default_value` only applies during CLI parsing; must set sensible values explicitly
- `forge_script::ScriptArgs` fields can be set via mutation without importing `EvmArgs`/`EnvArgs` types directly
- `foundry_config::Chain::from(u64)` creates a chain from EIP-155 chain ID
- `treb_config::ResolvedConfig` is re-exported at crate root; sender addresses are in `senders` HashMap as `Option<String>`
- Foundry script state machine: `ScriptArgs.preprocess() -> compile() -> link() -> prepare_execution() -> execute()`
- `foundry_evm::core::decode::decode_console_logs` decodes all console.log variants, silently skips unknown
- Foundry v1.5.1 uses `alloy-json-abi` v1.5.7 â€” use `alloy-json-abi = "1.5"` (not workspace v0.8) in treb-forge
- `ContractsByArtifact` from `foundry_common` implements `From<ProjectCompileOutput>` and `Deref<Target = BTreeMap<ArtifactId, ContractData>>`
- `ArtifactId.version` is `semver::Version` (no Default) â€” construct via `foundry_config::semver::Version::new()`
- `forge_script_sequence::BroadcastReader` wraps broadcast file reading; requires existing `broadcast/` dir
- `config.broadcast` (from `foundry_config::Config`) gives the broadcast root directory path
- `Config::load_with_root(path)` requires `.sanitized()` to resolve relative paths; without it, `src`, `test`, etc. resolve relative to CWD
- `foundry_common::version::SHORT_VERSION` contains the forge version and git commit at compile time
- Test fixtures with foundry.toml need `.gitignore` for `cache/` and `out/` build artifacts
