## Codebase Patterns
- Workspace root: `Cargo.toml` with `resolver = "3"`, edition 2024
- Crates live in `crates/<name>/` (e.g., `crates/treb-cli`, `crates/treb-core`)
- Use `workspace = true` inheritance for version, edition, rust-version, authors, license
- Nightly toolchain required for rustfmt nightly features (imports_granularity, wrap_comments, etc.)
- rustfmt.toml and clippy.toml follow foundry's conventions
- Cargo.lock is committed (binary crate in workspace)
- Foundry git deps pinned to tag v1.5.1; workspace deps defined in root `[workspace.dependencies]`
- svm-rs-builds has a duplicate SOLC_VERSION_0_8_31 bug; patched via `[patch.crates-io]` pointing to alloy-rs/svm-rs v0.5.24
- Use `//!` (inner doc comments) for crate-level docs, `///` for item docs — clippy enforces this
- Re-export foundry crates from treb-core with `pub use foundry_config; pub use foundry_common;`
- Error types live in `treb_core::error` module; re-exported as `treb_core::TrebError` and `treb_core::Result<T>`
- Use `thiserror` in library crates, `anyhow` in binary crates
- rustfmt merges `use super::X; use super::Y;` into `use super::{X, Y};` — write merged imports from the start
- CI: `dtolnay/rust-toolchain@nightly` + `Swatinem/rust-cache@v2`; 4 jobs: check, test, clippy, fmt
