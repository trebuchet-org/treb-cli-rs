## Codebase Patterns
- Use `{ workspace = true }` for all dependency declarations in crate Cargo.toml files
- Use `#[derive(Default)]` instead of manual `Default` impl when all fields have default values (clippy enforces this)
- All JSON-facing structs use `#[serde(rename_all = "camelCase")]` for Go compatibility
- Constants for file names go in `lib.rs`, types in `types.rs`, re-exports at crate root
- Run `cargo clippy -p <crate> -- -D warnings` — treats all warnings as errors
- Use `FileExt::unlock(&file)` (fully-qualified) instead of `file.unlock()` to avoid MSRV 1.85 vs std 1.89 conflict
- For atomic JSON writes: `io::write_json_file` handles parent dirs, temp file, persist, trailing newline
- For file locking: `io::with_file_lock(path, || { ... })` — pass data file path, lock file is auto-managed
- Store pattern: `new(registry_dir)` → `load()` → CRUD ops; each mutation calls `save()` internally with locking
- Golden file tests: load fixture via `serde_json::from_str`, insert each entry, re-read from disk, compare via `serde_json::Value` equality
- Fixtures live in `crates/treb-core/tests/fixtures/` — reference via `env!("CARGO_MANIFEST_DIR")` relative paths
- LookupStore is stateless (no in-memory data) — `rebuild(&deployments)` builds, saves, and returns the index
- Query methods on LookupIndex: `find_by_name`/`find_by_address` are case-insensitive, `find_by_tag` is exact match
- Registry facade: `Registry::open(root)` for existing, `Registry::init(root)` to create; deployment mutations auto-rebuild lookup index
- Use `match` instead of `unwrap_err()` when the Ok type doesn't derive Debug
