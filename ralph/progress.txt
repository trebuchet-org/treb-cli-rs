## Codebase Patterns
- Workspace crates follow pattern: `crates/<name>/Cargo.toml` with `version.workspace = true`, `edition.workspace = true`, etc.
- All shared deps are declared in root `[workspace.dependencies]` and referenced with `{ workspace = true }` in crate Cargo.toml
- Enum serde: use `#[serde(rename = "string_value")]` on each variant for string representation
- Enum pattern: derive `Clone, Debug, PartialEq, Eq, Hash, Serialize, Deserialize` + manual `Display` and `FromStr` impls
- Test pattern: serde round-trip tests iterate `(variant, expected_string)` tuples
- Error pattern: all treb errors use `TrebError` from `treb-core::error` with variants like `Config(String)`
- Type aliases: use `pub type Alias = OriginalType;` when two types share identical shape (e.g., `AccountConfig = SenderConfig`)
- File I/O modules: return `Result<T>` using `treb_core::error::Result`; use `TrebError::Config(String)` for parse/validation errors, and `TrebError::Io` (via `From<std::io::Error>`) for filesystem errors
- JSON output format: `serde_json::to_string_pretty` produces 2-space indentation; append `\n` for trailing newline to match Go format
- Test pattern for filesystem: use `tempfile::TempDir` for isolated temp directories in tests
- Env var tests: never share env var names between tests that run in parallel â€” use unique names or inline fixture TOML to avoid race conditions
- Format detection: inspect TOML line-by-line for section headers (`[accounts`, `[namespace`, `[ns.`) rather than parsing TOML
- Shared helpers: use `pub(crate)` visibility for internal helpers shared between modules (e.g., `expand_sender_config_env_vars`)
- V1 merge pattern: start with ns.default, overlay ns.<namespace> â€” senders use HashMap insert (namespace wins), profile/slow use if-let override
- Dotenv loading: use `dotenvy::from_path_override()` â€” silently skips missing files by ignoring the Result
- Foundry custom sections: `foundry_config::Config` doesn't preserve custom `[profile.*.treb.*]` sections â€” re-read foundry.toml as raw TOML via `toml::Value` and navigate manually
- Rust 2024 edition: `env::remove_var` is unsafe â€” wrap in `unsafe` block with SAFETY comment in tests
- Foundry config API: use `Config::load_with_root(project_root)` to specify project root, `config.root` gives the project root path
- Resolver pattern: `ResolveOpts` struct bundles all CLI/external overrides, `resolve_config` returns `ResolvedConfig`
- V2 namespace hierarchy: dot-walk "production.ntt" â†’ levels ["default", "production", "production.ntt"]; each level's profile/senders override previous
- Missing namespace detection: error if none of the non-default hierarchy levels exist as keys in config.namespace
