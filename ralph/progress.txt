## Codebase Patterns
- Workspace crates follow pattern: `crates/<name>/Cargo.toml` with `version.workspace = true`, `edition.workspace = true`, etc.
- All shared deps are declared in root `[workspace.dependencies]` and referenced with `{ workspace = true }` in crate Cargo.toml
- Enum serde: use `#[serde(rename = "string_value")]` on each variant for string representation
- Enum pattern: derive `Clone, Debug, PartialEq, Eq, Hash, Serialize, Deserialize` + manual `Display` and `FromStr` impls
- Test pattern: serde round-trip tests iterate `(variant, expected_string)` tuples
- Error pattern: all treb errors use `TrebError` from `treb-core::error` with variants like `Config(String)`
- Type aliases: use `pub type Alias = OriginalType;` when two types share identical shape (e.g., `AccountConfig = SenderConfig`)
