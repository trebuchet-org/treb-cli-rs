## Codebase Patterns
- `treb-cli` is a binary crate; `lib.rs` was created for the `ui` module to be testable. `main.rs` keeps `mod commands; mod output; mod ui;`.
- `TrebError::Cli(String)` variant added to `treb-core/src/error.rs` for CLI/interaction errors.
- `dialoguer` requires `features = ["fuzzy-select"]` for `FuzzySelect`; declared in workspace Cargo.toml.
- `fuzzy_select_deployment` uses a lifetime `'a` to tie the returned ref to the input slice.
- Tests for non-TTY behavior skip when `console::Term::stdout().is_term()` to avoid hanging in interactive sessions.
- `DeploymentStrategy` fields: `method`, `salt`, `init_code_hash`, `factory`, `constructor_args`, `entropy` (no `kind`/`factory_address`).
- `ArtifactInfo` fields: `path`, `compiler_version`, `bytecode_hash`, `script_path`, `git_commit`.
- `VerificationInfo` fields: `status`, `etherscan_url`, `verified_at`, `reason`, `verifiers: HashMap`.
