# Ralph Progress Log — Phase 07
Started: Tue Mar  4 2026

## Codebase Patterns
- Integration tests use `mod framework;` and `mod helpers;` to import shared test infrastructure
- Golden file tests follow: `TestContext::new("project")` → `IntegrationTest::new(name)` → `run_integration_test(&test, &ctx)`
- `seed_registry(ctx.path())` in a `post_setup_hook` seeds registry with 3 fixture deployments
- `PathNormalizer::new(vec![ctx.path().display().to_string()])` normalizes temp dir paths to `<PROJECT_ROOT>`
- `UPDATE_GOLDEN=1 cargo test` regenerates golden files from actual output
- New normalizers should NOT be added to default_chain() — use as extras via `.extra_normalizer()`
- Forge-specific normalizers: `CompilerOutputNormalizer`, `GasNormalizer`, `BlockNumberNormalizer`, `DurationNormalizer`
- Anvil-backed golden file tests use `#[tokio::test(flavor = "multi_thread")]` with `TestContext::new("project").with_anvil("anvil-31337").await`
- Fixture Deploy.s.sol must emit `ContractDeployed` and `TransactionSimulated` events for `treb run` to detect deployments
- `treb run` does NOT auto-detect raw CREATE ops — it only finds deployments via treb-specific events
- Error-path tests (no Anvil needed) use sync `#[test]`; Anvil-backed tests use `#[tokio::test(flavor = "multi_thread")]`
- `pre_setup_hook` removes `.treb/` + `foundry.toml` to simulate uninitialized projects for error tests

- **Learnings for future iterations:**

- **Learnings for future iterations:**
  - `--debug` flag in Forge enables interactive TUI debugger — in headless subprocess (no TTY), it passes through silently with no extra output

  - Error-path tests that don't need Anvil should use sync `#[test]` (not `#[tokio::test]`) per PRD requirement
  - Missing script errors manifest as "forge preprocessing failed" connection errors when no Anvil is running, since forge tries RPC before compilation
  - `pre_setup_hook` runs before setup commands — use it to delete `.treb/` and `foundry.toml` to simulate uninitialized state
