## Codebase Patterns
- TestWorkdir always creates `.treb/` directory; use `pre_setup_hook` with `remove_dir_all` to test uninitialized state
- Use fixed timestamps (`Utc.with_ymd_and_hms(...)`) in pre-populated test data for deterministic golden files
- Use `localhost` instead of `127.0.0.1` in test URLs to avoid VersionNormalizer mangling IPs
- Fork commands that should error on uninitialized projects need `ensure_treb_dir(&treb_dir)?` call
- Table timestamp format `%Y-%m-%d %H:%M:%S UTC` is partially normalized: date becomes `<DATE>`, time stays fixed
- JSON timestamp (ISO-8601) is fully normalized to `<TIMESTAMP>` by TimestampNormalizer
- Chain multiple `.test()` calls in IntegrationTest to verify state after mutations (e.g., exit then status)
- `with_context()` on errors produces "Caused by:" chains in golden file output — plan for multi-line error formats
