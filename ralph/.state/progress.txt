# Ralph Progress Log
Started: Tue Mar  4 2026

## Codebase Patterns
- `generate_deployment_id()` lives in `treb-core/src/types/ids.rs` — the single source of truth for deployment ID format
- `treb-forge/src/pipeline/hydration.rs` re-exports it via `pub use` for backward compatibility with existing `pipeline::generate_deployment_id` references
- Deployment ID format: `namespace/chainId/ContractName` (empty label) or `namespace/chainId/ContractName:label` (non-empty)
- Transaction ID format: `tx-0x<full_64_hex>` — the `tx-` prefix is prepended in hydration.rs (`format!("tx-{:#x}", ...)`) and register.rs (`format!("tx-{tx_hash}")`)
- All cross-references to transaction IDs (safe_transaction.transaction_ids, governor_proposal.transaction_ids) must also use the `tx-` prefix for populate_safe_context matching
- Artifact path uses `ArtifactId.source` (relative source path like "src/Counter.sol"), NOT `ArtifactId.path` (compiled output path like "out/Counter.sol/Counter.json")
- Golden file tests: regenerate with `UPDATE_GOLDEN=1 cargo test`; HashNormalizer auto-handles `tx-0x<HASH>` (matches `0x[0-9a-fA-F]{64}` inside the string)
