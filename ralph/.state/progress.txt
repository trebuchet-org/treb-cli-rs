## Codebase Patterns
- anvil::NodeHandle implements Drop that fires a graceful shutdown signal, but server JoinHandles are NOT aborted on NodeHandle drop — must store AbortHandles and call .abort() explicitly to free ports
- AbortHandle does NOT call abort on its own drop — explicit abort() is required in a Drop impl
- AnvilConfig builder pattern: struct fields with builder methods, spawn() async method returning AnvilInstance
- anvil::try_spawn returns (EthApi, NodeHandle); EthApi has all RPC methods (evm_snapshot, evm_revert, anvil_set_code, get_code, etc.)
- Alloy crates must be pinned to v1.1.1 via `[patch.crates-io]` to match foundry v1.5.1. Without this, cargo resolves alloy to 1.7.x which breaks alloy-evm 0.24.2 and the anvil crate.
- All core types use `#[serde(rename_all = "camelCase")]` for JSON serialization
- Type modules go in `crates/treb-core/src/types/` and are registered in `mod.rs` with pub use re-exports
- Error variants follow pattern `#[error("category error: {0}")] Category(String)` in TrebError
- Registry file constants go in `crates/treb-registry/src/lib.rs`
- Store modules follow pattern: struct with `path: PathBuf` + `data: T`, `new(registry_dir)`, `load()`, `save()` using `crate::io` helpers with `with_file_lock` for atomic writes
- New store modules go in `crates/treb-registry/src/store/` and are registered in `mod.rs` with re-exports in both `store/mod.rs` and `lib.rs`
