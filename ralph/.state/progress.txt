## Codebase Patterns
- Integration tests use `mod framework;` and `mod helpers;` to import shared test infrastructure
- Golden file tests follow: `TestContext::new("project")` ŌåÆ `IntegrationTest::new(name)` ŌåÆ `run_integration_test(&test, &ctx)`
- `seed_registry(ctx.path())` in a `post_setup_hook` seeds registry with 3 fixture deployments
- `PathNormalizer::new(vec![ctx.path().display().to_string()])` normalizes temp dir paths to `<PROJECT_ROOT>`
- `UPDATE_GOLDEN=1 cargo test` regenerates golden files from actual output
- New normalizers should NOT be added to default_chain() ŌĆö use as extras via `.extra_normalizer()`
- Forge-specific normalizers: `CompilerOutputNormalizer`, `GasNormalizer`, `BlockNumberNormalizer`, `DurationNormalizer`
- Anvil-backed golden file tests use `#[tokio::test(flavor = "multi_thread")]` with `TestContext::new("project").with_anvil("anvil-31337").await`
- `effective_rpc_url` in run.rs is the network NAME (e.g., "anvil-31337"), not the actual URL ŌĆö resolve via `treb_config::load_foundry_config` + `rpc_endpoints` for RPC calls
- Fixture Deploy.s.sol must emit `ContractDeployed` and `TransactionSimulated` events for `treb run` to detect deployments
- `treb run` does NOT auto-detect raw CREATE ops ŌĆö it only finds deployments via treb-specific events
- Error-path tests (no Anvil needed) use sync `#[test]`; Anvil-backed tests use `#[tokio::test(flavor = "multi_thread")]`
- `pre_setup_hook` removes `.treb/` + `foundry.toml` to simulate uninitialized projects for error tests
- `treb run --broadcast` does NOT send real Ethereum transactions to Anvil ŌĆö it simulates in-process via forge pipeline; use `eth_sendTransaction` directly for real on-chain txs
- For `treb register` tests, deploy contracts directly on Anvil via `reqwest` + `eth_sendTransaction` RPC call, then register by tx hash
- Happy-path register tests need dynamic tx hash ŌåÆ use `GoldenFile` directly instead of `IntegrationTest` builder
