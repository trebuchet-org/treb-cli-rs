## Codebase Patterns
- Rust `regex` crate does NOT support look-around (look-ahead/look-behind); use word boundaries `\b` or capture groups instead
- Framework modules registered in `crates/treb-cli/tests/framework/mod.rs`
- Workspace-level deps added in root `Cargo.toml` under `[workspace.dependencies]`, then referenced in crate `Cargo.toml` with `{ workspace = true }`
- `similar` v2.7.0 `TextDiff::from_lines()` requires both args to be the same type — use `.as_str()` on `String` to match `&str`
- Avoid `env::set_var`/`env::remove_var` in parallel tests (Rust 2024 makes them `unsafe` + they race); parameterise with a flag instead
- Async cleanup in sync Drop: `tokio::task::block_in_place` + `Handle::current().block_on()` (requires `multi_thread` runtime)
- `Semaphore::acquire().forget()` + `add_permits(1)` for manual permit management in pool patterns
