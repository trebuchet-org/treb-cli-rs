# Ralph Progress Log — Phase 07

## Codebase Patterns
- Integration tests use `mod framework;` and `mod helpers;` to import shared test infrastructure
- Golden file tests follow: `TestContext::new("project")` → `IntegrationTest::new(name)` → `run_integration_test(&test, &ctx)`
- `seed_registry(ctx.path())` in a `post_setup_hook` seeds registry with 3 fixture deployments
- `PathNormalizer::new(vec![ctx.path().display().to_string()])` normalizes temp dir paths to `<PROJECT_ROOT>`
- `UPDATE_GOLDEN=1 cargo test` regenerates golden files from actual output
- New normalizers should NOT be added to default_chain() — use as extras via `.extra_normalizer()`
- Forge-specific normalizers: `CompilerOutputNormalizer`, `GasNormalizer`, `BlockNumberNormalizer`, `DurationNormalizer`

## 2026-03-04 - P7-US-001
- Added 4 new normalizers to `crates/treb-cli/tests/framework/normalizer.rs`:
  - CompilerOutputNormalizer: normalizes `Compiling N files with solc X.Y.Z` and `Solc X.Y.Z finished` lines
  - GasNormalizer: normalizes gas values (`gas: NNN`, `Gas used: NNN`, `gasUsed: NNN`)
  - BlockNumberNormalizer: normalizes block numbers (`block: N`, `blockNumber: N`, `Block Number: N`)
  - DurationNormalizer: normalizes duration strings (`1.23s`, `456ms`, `2µs`)
- 8 new unit tests added (2 per normalizer: match case + no-match case), CompilerOutputNormalizer has 3 tests
- NOT added to default_chain() — designed as extras for forge-output-heavy tests
- Files changed: `crates/treb-cli/tests/framework/normalizer.rs`
- **Learnings for future iterations:**
  - Regex `(?i)` flag handles case-insensitive matching for forge output variations (Solc vs solc)
  - Gas/block patterns use `(?i)(prefix)\s*:\s*\d+` → `${1}: <PLACEHOLDER>` to preserve the original prefix casing
  - Duration pattern `\d+(?:\.\d+)?(?:µs|ms|s|m)\b` covers all common forge timing formats
