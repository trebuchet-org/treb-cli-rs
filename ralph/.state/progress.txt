## Codebase Patterns
- All normalizers live in `crates/treb-cli/tests/framework/normalizer.rs` and implement the `Normalizer` trait
- Each normalizer uses `Regex::new(...).unwrap()` inline (no lazy_static) for simplicity
- Unit tests follow pattern: one `_replaces_*` test for match cases, one `_no_match` test for non-match cases
- Normalizer structs are unit structs unless they need config (like PathNormalizer with stored paths)
- Changing the default chain or NormalizerChain post-processing requires regenerating ALL golden files: `UPDATE_GOLDEN=1 cargo test --workspace`
- `NormalizerChain::normalize()` applies post-processing (CRLFâ†’LF, trim, trailing newline) on ALL chains, not just default
- Golden files: command output â†’ `commands.golden`, each artifact â†’ `{stem}.golden` (file_stem of artifact path)
- `DEFAULT_OUTPUT_ARTIFACTS` holds bare filenames; `with_default_artifacts()` prepends `.treb/`
- 3 tests bypass IntegrationTest framework and use raw `GoldenFile::compare` with `"output"` case name
